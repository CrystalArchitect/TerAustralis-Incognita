# CrystalCore — one name, five things

"CrystalCore" is the project's oldest and most overloaded name. This page is
the disambiguation map; every use of the name in the repo should resolve to
exactly one row here.

| Name as used | What it is | Layer | Where |
|---|---|---|---|
| **CrystalCore framework** | The sovereign-companion framework Lumina runs on: brain, layered memory, profiles | Built | `src/apps/lumina/crystalcore/` |
| **crystal-core (protocol pack)** | Starline Weaver, Decode→Ingest→Twin pipeline, Starline, RDP record kernel | Built | `src/crystal-core/` |
| **crystalcore (CrystalBridge)** | The MCP consent gate — a Python package that happens to share the name | Built | `src/crystalcore/` |
| **CrystalCore.OS (mythos terminal)** | The mythos as a terminal you can fly — a playable story, not an operating system | Vision | `src/crystalcore-os/crystalcore_os.py` + [`mythos/CRYSTALCORE-OS.md`](../../mythos/CRYSTALCORE-OS.md) |
| **CrystalCore OS (platform architecture)** | This repository's organizational architecture — the directory layout, documentation tree, governance, and AI collaboration model | Process | [`docs/architecture/SystemMap.md`](../architecture/SystemMap.md), [`ADR-0001`](../adr/ADR-0001.md) |

Three of these are running, tested code. The mythos terminal is a story with
a command line. The platform architecture is neither software nor story — it
is how the repository itself is organized and governed.

Per the [Incognita Rule](../governance/The-Incognita-Rule.md), the mythos
terminal may *point at* the real components and say "that one is real"; it
never borrows their authority. And the platform architecture, despite the
"OS" in its name, runs nothing — it organizes.

The name itself is inspired by the ordered structures of crystallography:
building with clarity, stability, and user-defined structure.
