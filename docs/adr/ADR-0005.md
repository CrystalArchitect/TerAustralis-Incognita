# ADR-0005: AI Orchestrator — consolidate the naming; ship the concept as documentation first

**Status:** Accepted
**Date:** 2026-07-23

## Context

Two independent sketches of the same responsibility appeared in the same
round of review: an "AI Router" module inside a proposed CrystalCore
Engine, and an "AI Orchestrator" sitting in front of the six documented AI
roles ([`docs/ai/`](../ai/)) to recommend which tool fits a given task. Both
describe the same job — decide which AI handles a task — under different
names, which is exactly the kind of ambiguity [`ADR-0004`](ADR-0004.md)
exists to prevent.

Separately, there was a real question of how much to build: an automated,
always-on router is a new piece of infrastructure with its own failure
modes (what happens when it recommends wrong, or nobody's watching); a
human reading a table has none of those, at some cost in convenience.

## Decision

- **One name: AI Orchestrator.** "AI Router" is retired as a separate
  concept; anywhere routing-between-AIs comes up, this is the name for it
  ([`docs/ai/AI-Architecture.md`](../ai/AI-Architecture.md)).
- **Define it as recommend-then-human-decides, not autonomous dispatch.**
  The internal shape is Task → Capability Assessment → Recommended AI →
  Human Review — the human keeps the final call at every step, the same
  veto this project holds everywhere else
  ([`The-Incognita-Rule.md`](../governance/The-Incognita-Rule.md) §3: no
  line mints its own authority).
- **Ship the first increment as documentation, not code.**
  [`docs/ai/Decision-Matrix.md`](../ai/Decision-Matrix.md) — a task-type →
  recommended-AI → human-review-level table — *is* that first increment,
  not a placeholder for a future one. No runtime, no automation, no new
  failure mode, and no maintenance burden beyond keeping a markdown table
  current.
- **Automating the middle step (an actual capability-assessment routine)
  is out of scope here.** If it's ever built, it needs its own spec and its
  own ADR before implementation — the same discipline
  [`ADR-0001`](ADR-0001.md) followed for the repository reorganization
  itself.

## Consequences

- Anyone asking "which AI for this task" today has an answer
  (`Decision-Matrix.md`) that exists now, not after a future build.
- No coordination code exists to maintain, secure, or debug — the concept's
  entire footprint today is documentation, which fails safe by construction
  (a stale table gives a bad recommendation a human can override; it can't
  silently misroute anything on its own).
- If usage shows the table is consulted often enough that automating the
  lookup would help, that's the trigger for a follow-up spec — not a
  deadline this ADR sets.
