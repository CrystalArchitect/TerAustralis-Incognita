# ADR-0001: Adopt the CrystalCore OS v1.0 repository architecture

**Status:** Accepted
**Date:** 2026-07-23

## Context

The repository grew by accretion — three merged repos, code and
documentation and lore interleaved at the root (five code directories, four
governance docs, two content universes, an archive with a leading
underscore). The maintainer, working with ChatGPT as systems architect,
specified the "CrystalCore OS v1.0" blueprint: a conventional, legible
layout separating vision, architecture, governance, research, engineering,
archive, and AI collaboration — with Claude designated to implement it.

Constraints: CI, the Pages deploy, `__file__`-anchored code, the site's
content copies, the Constitution's amendment rules, and the Incognita Rule
(the new docs must not claim more than exists).

## Decision

Reorganize to the blueprint, preserving history via `git mv`. The mapping:

| Was | Now |
|---|---|
| `apps/`, `crystal-core/`, `crystalcore/`, `node/`, `sdk/`, `site/`, `profiles/` | `src/<same name>/` (uniform shift — ADR-0003) |
| `mythos/crystalcore_os.py` | `src/crystalcore-os/` (code leaves the content area) |
| `crystal-core/spec/*`, `clementine/{CLEMENTINE,STARLINE-WEAVE-PROTOCOL}.md`, `rdp/RDP-INTEGRATION.md` | `docs/architecture/crystal-core/` |
| seven-sisters cycle docs, water briefs, transmit records (from `crystal-core/`) | `research/seven-sisters/` |
| `_archive/{crystalcore-app, local-snapshot-2026-07-17}` | `archive/{legacy/…, 2026/…}` |
| `TeraAustralis/` (lore, publish) | `mythos/teraaustralis/` (ADR-0002) |
| `TeraAustralis/strategy/southern-pillar.md` | `docs/vision/SouthernPillar.md` |
| `CrystalCore.Lattice/activation/BOOT_STATUS.md` | `docs/architecture/lattice/BOOT_STATUS.md` |
| `CONSTITUTION.md`, `ROADMAP.md`, `THE-INCOGNITA-RULE.md` | `docs/governance/{Constitution,Roadmap,The-Incognita-Rule}.md` |
| `GITHUB-COMMIT-INSTRUCTIONS.md`, `docs/{ACCESS,MCP-GUEST,PUSH}.md` | `docs/guides/` |
| `docs/FULL-STACK-v0.5.md` | `docs/architecture/Full-Stack-v0.5.md` |
| `tests/test_mesh_stub.py` | `tests/unit/` |
| `crystal-core/.github/ISSUE_TEMPLATE/{bug_report,feature_request}.md` | `.github/ISSUE_TEMPLATE/` (nested `.github/` was inert; its boilerplate `custom.md` and dead `static.yml` deleted) |

New material: the documentation tree (`docs/{vision,architecture,governance,
ai,agents,guides,adr}/`), `.github/` scaffolding (PR/discussion templates,
CODEOWNERS), `scripts/maintenance/check.sh`, area READMEs, `CHANGELOG.md`.
Every reference to a moved path — workflows, `pyproject.toml`,
`.gitignore`, docstrings, markdown links, the site's content copies — was
updated in the same change; CI runs the same checks against the new paths
plus the previously-unwired `tests/` suite.

Root keeps only: licenses/NOTICE, README, CONTRIBUTING, CODE_OF_CONDUCT,
SECURITY, AGENTS.md (tool convention), CHANGELOG, CNAME (Pages), and the
Python packaging files.

## Consequences

- One place per kind of material; newcomers can navigate from
  `docs/architecture/SystemMap.md`.
- External links into moved files (old blob URLs) break — accepted;
  history survives via rename detection.
- The Constitution's path references changed, carried by an amendment-log
  entry that takes force when the maintainer merges (§8).
- Deep-tree relative links are easier to get wrong — mitigated by the
  sweep being verified with self-tests, suites, and a site build before
  merge.
