# ADR-0003: Move code into src/ as a uniform shift; keep runtime-coupled files with their code

**Status:** Accepted
**Date:** 2026-07-23

## Context

The blueprint's rule is "src/ contains only executable code." Three
mechanics questions had to be settled: how to move seven top-level
directories without breaking the code's own path math, what to do with
runtime configuration, and which documentation-shaped files must *not*
leave the source tree.

The path math is the sharp edge: `crystalcore/config.py` anchors
`profiles/` two parents up from `__file__`, and `crystalcore/bridge.py`
reaches `apps/lumina/crystalcore/` the same way. Any move that changes the
*relative* geometry between these directories changes runtime behavior.

## Decision

- **Uniform one-level shift.** `apps/`, `crystal-core/`, `crystalcore/`,
  `node/`, `sdk/`, `site/`, and `profiles/` all move into `src/` unchanged.
  Every `__file__`-relative relationship is preserved; the only Python
  change is renaming the (now inaccurate) `REPO_ROOT` constant to
  `SRC_ROOT` and updating docstrings. `pyproject.toml` adopts the src
  layout (`packages.find.where = ["src"]`, pytest `pythonpath = ["src"]`).
- **`src/profiles/` stays beside the code that anchors to it.** It is
  runtime configuration (committed `bridge_config.json`; gitignored audit
  and message logs), and co-location means zero behavioral change. The
  `.gitignore` patterns moved with it.
- **Runtime-coupled and component-local files stay in `src/`**:
  `clementine/transcripts/` (the bridge *writes* there at runtime),
  `services/sample-events/` and `rdp/vectors.json` (test fixtures),
  component `README.md`s (navigation, kept beside their code), and
  `requirements-*.txt` (installed by CI relative to the component).
- **Standalone specs leave `src/`** for `docs/architecture/crystal-core/`
  (`ARCHITECTURE`, `BLUEPRINT-v0.3`, `STARLINE`, `CLEMENTINE`,
  `STARLINE-WEAVE-PROTOCOL`, `RDP-INTEGRATION`), with pointers updated in
  the docstrings and READMEs that cite them.

## Consequences

- The bridge, gate, and Lumina loader behave identically before and after —
  verified by the self-tests and suites rather than asserted.
- Anyone running the bridge from source must put `src/` on `PYTHONPATH`
  (or pip-install the package); `docs/guides/MCP-Guest.md` shows the
  updated invocation.
- "Docs live in `docs/`" now has explicit, recorded exceptions — READMEs
  and fixtures — so the rule can be enforced without litigating them again.
