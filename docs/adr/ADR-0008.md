# ADR-0008: Supersede ADR-0006 §1 — adopt CC BY-NC-ND 4.0 for code; reconcile the fallout

**Status:** Accepted — supersedes [ADR-0006](ADR-0006.md) Decision §1 only
**Date:** 2026-07-23

## Context

[`ADR-0006`](ADR-0006.md) decided to keep Apache-2.0 for code, reasoning
through it as a deliberate open-core trade-off. Independently, in a
separate, uncoordinated Claude session, the maintainer merged a PR that
changed `LICENSE` from Apache-2.0 to CC BY-NC-ND 4.0 "for commercial
exclusivity" — full non-commercial-only terms on the code, not just the
content. The maintainer has since confirmed, directly, that this is the
intended direction: keep CC BY-NC-ND 4.0 for code.

That PR changed four files (`LICENSE`, `NOTICE`, `CONTRIBUTING.md`,
`docs/ATTRIBUTIONS.md`) but left roughly a dozen other places in the
repository still asserting the code is Apache-2.0 — two nested per-component
`LICENSE` files with the full old text, a live site footer component, an
npm package manifest, mythos boilerplate footers, a governance doc, and
`docs/ATTRIBUTIONS.md`'s own later sections contradicting its earlier ones
(it called the code Apache-2.0 in its "License Enforcement" and "Rebranding
& Theft Prevention" sections while correctly calling it CC BY-NC-ND
elsewhere — those sections literally told readers "you can use it freely
... for commercial ... projects," the opposite of the actual intent). The
same PR also silently reverted [`ADR-0007`](ADR-0007.md)'s fix — `NOTICE`
was rewritten from scratch and dropped "TerAustralis Incognita" and the
named copyright holder entirely, replacing them with a generic "CrystalCore
OS Contributors" line, because that session didn't know ADR-0007 existed.

Two structural questions needed answering before this could just be
"apply the same find-and-replace everywhere": whether relicensing
retroactively affects anyone besides the maintainer, and whether a strict
No-Derivatives license is even compatible with the pull-request contribution
model this repository's own `CONTRIBUTING.md` describes.

## Decision

- **ADR-0006 §1 is superseded.** The code-licensing choice is now CC BY-NC-ND
  4.0, full commercial exclusivity, not the open-core Apache-2.0 trade ADR-0006
  chose. ADR-0006's §2 (six licensing/IP principles) and §3 (trademark status)
  are unaffected and still stand — this ADR does not touch them. ADR-0006's
  own text is left in place as the historical record of the option
  considered and why, per this repo's own convention for a decision that
  revises an earlier one; only its Status line and a short inline note point
  here.
- **Relicensing consent checked, not assumed.** `git log` shows exactly two
  commit authors in this repository's entire history: Claude (acting as the
  maintainer's tool) and Crystal Arena-Turner (the maintainer, matching
  `NOTICE`'s copyright holder and the ABN from [`ADR-0007`](ADR-0007.md)). No
  third-party contributor has ever committed code here, so there is no
  outstanding consent gap today. This is stated as a verified fact, not an
  assumption — if that ever changes, it changes because someone else's PR
  gets merged, which is exactly the case the next bullet covers.
- **Added a contribution-licensing clause to `CONTRIBUTING.md`** resolving
  the literal conflict between "no derivative redistribution" and "propose
  changes via pull request": contributing a PR grants the maintainer rights
  broader than the public CC BY-NC-ND terms (standard "inbound ≥ outbound"
  structure), so merging contributions doesn't itself violate the license
  the repository ships under. This is a lighter-weight substitute for a
  formal CLA — worth revisiting as an actual signed CLA if outside
  contribution volume grows past occasional PRs.
- **`docs/` staying inside the CC BY-NC-ND scope (not just `src/`) is
  confirmed, not just left as a default.** The maintainer explicitly
  confirmed documentation should carry the same non-commercial,
  no-derivatives terms as code — `CONTRIBUTING.md`'s licensing section
  already covers `src/`, `docs/`, `tests/`, and `scripts/` together, and
  that scope is now a deliberate decision, not an artifact of PR #32's
  wording.
- **Fixed every place found still claiming Apache-2.0 for code**, to bring
  the repository back to internal consistency:
  - `NOTICE` — restored "TerAustralis Incognita" and the named copyright
    holder that the other session's rewrite dropped, without touching the
    CC BY-NC-ND terms it correctly wrote.
  - `docs/ATTRIBUTIONS.md` — rewrote the self-contradicting "License
    Enforcement" and "Rebranding & Theft Prevention" sections, which had
    told readers commercial use was freely permitted.
  - Two nested per-component `LICENSE` files with full Apache-2.0 text
    (`src/crystal-core/LICENSE`, `src/apps/crystal-interface/LICENSE`)
    **deleted** rather than rewritten — a monorepo with one root `LICENSE`
    doesn't need per-component duplicates that can drift again, and no
    other component had one.
  - `README.md`, `LICENSE-CONTENT.md`, `mythos/README.md`,
    `mythos/COVENANT.md`, `mythos/content/GOVERNANCE.md` and its
    `src/site/` mirror, `mythos/content/LICENSE-CONTENT.md` and its
    `src/site/` mirror, `docs/governance/Development-Standards.md`,
    `docs/architecture/crystal-core/Crystal-Runtime-Specification-v0.3.md`,
    `src/crystal-core/README.md`, `src/apps/crystal-interface/README.md` —
    prose corrected to CC BY-NC-ND 4.0.
  - `src/site/src/lib/components/Footer.svelte` — the live site's own
    footer was telling visitors the code was Apache-2.0 with a link to
    apache.org; corrected, since this one was actually public-facing.
  - `src/sdk/typescript/package.json`'s `license` field — changed to
    `CC-BY-NC-ND-4.0` for consistency. The maintainer confirmed this is
    intended: the SDK is not meant to be more freely reusable than the rest
    of the codebase, unusual as No-Derivatives is for a published npm
    package.
  - MemClaw's own attribution (Apache 2.0, referenced concepts only, not
    incorporated code) is unaffected throughout — only claims about
    *this* project's own license were wrong.

## Consequences

- The repository no longer says two different things about its own code
  license depending on which file a reader opens. Before this ADR, a reader
  could hit "Apache-2.0" or "CC BY-NC-ND 4.0" for the same code depending on
  which of ~15 files they read; now it's one answer everywhere except the
  historical record in `ADR-0006` §1, which is explicitly marked as such.
- Code is now fully commercially exclusive: no patent grant (CC BY-NC-ND has
  none, unlike Apache-2.0 §3), and no permissive fallback for anyone —
  competitor or well-meaning non-commercial user — to build on modified
  versions. That's the accepted cost of the maintainer's actual choice, not
  an oversight.
- `NOTICE`'s accuracy is restored; the collision between this decision and
  ADR-0007 is closed.
- The `docs/` scope and the npm SDK license field were both raised as
  explicit questions rather than silently resolved, and the maintainer
  confirmed both as intended — CC BY-NC-ND applies to `docs/` as well as
  code, and the SDK is not an exception.
- If a third-party contributor ever does submit a PR, the new
  `CONTRIBUTING.md` clause is what makes merging it lawful under the
  project's own terms — it needs to actually exist in the contributor's
  view (i.e., stay in `CONTRIBUTING.md`, not get quietly dropped the way
  `NOTICE` was) for that to keep holding.
