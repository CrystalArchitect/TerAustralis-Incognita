# ADR-0009: Reconcile the licensing chaos — CC BY-NC-ND 4.0 governs today; packages/ is an in-progress, not-yet-authoritative target

**Status:** Accepted
**Date:** 2026-07-23

## Context

Within roughly 45 minutes, at least three uncoordinated Claude sessions and
one direct push from the maintainer landed licensing-related changes on
`main`, none aware of the others' work:

1. This session wrote [`ADR-0006`](ADR-0006.md) (keep Apache-2.0 for code),
   then merged it.
2. A separate session changed root `LICENSE` to CC BY-NC-ND 4.0 "for
   commercial exclusivity," which this session then reconciled as
   [`ADR-0008`](ADR-0008.md) (supersedes ADR-0006 §1) — fixing roughly
   fifteen files still claiming Apache-2.0.
3. A third session (different from both above) independently filled in the
   Apache-2.0 copyright holder in `LICENSE`, then added Apache-2.0
   copyright/SPDX headers to 97 source files under `src/`, then discovered
   `main` actually said CC BY-NC-ND and rewrote `LICENSE` to match —
   without updating the 97 headers it had just added, leaving them
   asserting Apache-2.0 against a CC BY-NC-ND file.
4. The maintainer (or a fourth actor, committed under the maintainer's own
   git identity) pushed a large restructuring — `packages/`, seven
   independent Python packages migrated out of `src/`, each with its own
   `LICENSE.md`: AGPL v3 for most, Proprietary for CrystalBridge, a
   MIT/Commercial dual license for CrystalCore-EI (`LICENSE.md` +
   `COMMERCIAL_LICENSE.md` + `LICENSING.md` at the repo root, all titled
   ambiguously as if they were repo-wide), CC BY-NC-ND for mythos. The
   commit message says "Ready for Phase 4 (test and publish)" — `src/` was
   not deleted (3 lines removed across a 14,585-line commit), and the root
   `pyproject.toml` still points at `src/`, meaning `packages/` is a
   parallel scaffold, not yet the running code.

The maintainer asked for a side-by-side of the resulting five contradictory
descriptions of the project's license, reviewed it, and asked to fix what's
actually wrong and push — without picking between "uniform CC BY-NC-ND" and
"differentiated per-package" as the long-term target, since that migration
is explicitly unfinished.

## Decision

- **What governs today: root `LICENSE` (CC BY-NC-ND 4.0), for `src/` and
  mythos alike.** This is not a new choice — it's recognizing that `src/`
  is what `pyproject.toml` actually builds and what CI actually tests, and
  the root `LICENSE` file governs it. This ADR does not reopen ADR-0008.
- **`packages/` and its differentiated per-package licensing
  (`docs/governance/LICENSING-STRATEGY.md`) is the migration's declared
  *target*, not today's license.** It's real, detailed, deliberate work —
  not discarded or overridden — but it isn't wired in (no `pyproject.toml`
  changes, `src/` still present and canonical), so this ADR doesn't treat
  it as already in effect. `README.md` now says this explicitly instead of
  picking one of the two silently.
- **Fixed the concrete, unambiguous bugs** — errors regardless of which
  long-term direction wins:
  - `README.md`'s license section claimed "Core systems: Apache-2.0,"
    which matched *neither* the root `LICENSE` (CC BY-NC-ND) *nor* the
    actual `packages/*/LICENSE.md` files (AGPL v3) it was supposedly
    summarizing. Rewritten to state what's actually true: `src/` is
    CC BY-NC-ND today; `packages/` is the in-progress target.
  - **97 source files' SPDX headers** (`src/apps/`, `src/crystal-core/`,
    `src/crystalcore/`, `src/crystalcore-os/`, `src/node/`, `src/sdk/`,
    `src/site/` — added by session 3, self-corrected in `LICENSE` but not
    in the headers) said `SPDX-License-Identifier: Apache-2.0`. Batch
    -corrected to `CC-BY-NC-ND-4.0`, matching the file they actually live
    under (`src/`, governed by root `LICENSE`).
  - **`LICENSE.md`, `LICENSING.md`, `COMMERCIAL_LICENSE.md`** (new root
    files from the restructuring) read as if they described the whole
    repository's license — "CrystalCore.OS Dual-Licensing Model" — but
    their actual content (MIT non-commercial + paid commercial, pricing
    tiers, API-call volume) matches specifically
    `packages/crystalcore-ei/LICENSE.md`. Added a scope note to each
    pointing at the CrystalCore-EI package and at this ADR, so a reader
    doesn't mistake them for the repo's real license the way `README.md`
    briefly did.
- **Merged the restructuring into this branch** rather than resolving the
  conflict by discarding either side — the only real content conflict was
  `README.md`'s license bullet (resolved above); the SPDX-header commit and
  the `packages/` commit touched disjoint files and merged cleanly.

## Consequences

- Anyone asking "what license is this repository under right now" gets one
  consistent answer — CC BY-NC-ND 4.0, governing `src/` and `mythos/` alike
  — instead of five documents disagreeing, three of them by accident.
- The `packages/` restructuring and its differentiated licensing plan are
  preserved intact, not overwritten — they're labeled as in-progress
  instead of silently treated as either already-true or discarded.
- **The actual decision this ADR does not make**: whether the repository
  ultimately converges on uniform CC BY-NC-ND (today's answer) or the
  differentiated per-package model (`packages/`'s target) is still open.
  That's a substantive strategic call, already asked about twice in this
  conversation; resolving it here a third time by implication would repeat
  the exact failure mode this ADR exists to fix. It becomes real the day
  `pyproject.toml` (or equivalent) actually points at `packages/` instead
  of `src/` — that's the trigger for the ADR that finally picks one.
- If another uncoordinated session touches licensing again before that
  happens, this ADR is the one place documenting why five different
  answers existed and which one was live at the time — worth checking
  before adding a sixth.
