# ADR-0007: Correct the project name to "TerAustralis Incognita"; rename mythos/teraaustralis/

**Status:** Accepted
**Date:** 2026-07-23

## Context

[`ADR-0006`](ADR-0006.md) recorded the project's licensing/IP principles and,
in doing so, repeated a spelling — "TeraAustralis Incognita" (two a's) —
that turns out not to match the maintainer's actual registered trading
name. The maintainer confirmed the ABN's trading name is **TerAustralis
Incognita** (one 'a').

A repository-wide check found the root legal files already agree with the
ABN, not with the drifted spelling:

- `README.md`'s title: `# TerAustralis Incognita`
- `NOTICE` contradicted itself — line 1 ("TeraAustralis Incognita") and
  line 2 ("Copyright 2026 Crystal Arena-Turner (**TerAustralis** Incognita)")
  disagreed within the same file.
- Multiple files under `archive/` (untouched, historical) consistently pair
  **ABN 70 741 068 059** with **"TerAustralis Incognita"**, e.g.
  `archive/legacy/crystalcore-app/apocryphon.html`: *"ABN 70 741 068 059.
  © 2026 TerAustralis Incognita."*

The double-a spelling entered the repository during the v1.0
reorganization ([`ADR-0001`](ADR-0001.md), [`ADR-0002`](ADR-0002.md)),
which moved the old `TeraAustralis/` content folder to
`mythos/teraaustralis/` — carrying the drift into a directory name, and
from there into every doc that referenced the new path. It was never a
deliberate rename; nothing in either ADR discusses the spelling as a
choice.

## Decision

- **Canonical spelling: "TerAustralis Incognita"** (one 'a'), matching the
  registered ABN trading name. This is the name for the project going
  forward.
- **Directory renamed**: `mythos/teraaustralis/` → `mythos/teraustralis/`
  via `git mv`, preserving history. Every current-state reference to that
  path is updated to match (`README.md`, `AGENTS.md`, `LICENSE-CONTENT.md`,
  `docs/architecture/SystemMap.md`, the two `ARCHITECTURE.md` copies,
  `docs/vision/SouthernPillar.md`, `docs/governance/Constitution.md` §7).
- **Prose references corrected** everywhere the name appears as live,
  current-state text: `README.md`, `NOTICE`, `CONTRIBUTING.md`, `AGENTS.md`,
  `docs/vision/Mission.md`, `docs/governance/Constitution.md` §1 (a locked
  name — corrected via the Constitution's own §8 amendment process, see its
  amendment log), `ADR-0006.md`, `mythos/README.md`, the sibling-repo
  blurbs in `src/crystal-core/README.md` and
  `src/apps/crystal-interface/README.md`, and the content files that moved
  with the directory (`mythos/teraustralis/**`).
- **What's deliberately left alone**:
  - **`archive/`** — never edited, per
    [`Repository-Principles.md`](../governance/Repository-Principles.md) #3
    ("preserve historical material"). It already has the correct spelling
    throughout, which is itself the evidence this ADR relies on.
  - **`ADR-0001.md` and `ADR-0002.md`** — left as the unedited historical
    record of what was actually decided and done in the v1.0 reorg,
    including its drifted spelling. This ADR supersedes that one detail
    without rewriting their text — the standard pattern for a decision that
    revises an earlier one.
  - **`CHANGELOG.md` and `Roadmap.md`'s "Recently landed" entries** — dated
    log entries describing what happened at the time; corrected with a new
    dated entry each rather than rewritten in place.
  - **`docs/vision/SouthernPillar.md`'s provenance line** ("moved here from
    `TeraAustralis/strategy/southern-pillar.md`") — describes the old,
    pre-reorg path that no longer exists; left as historical provenance.
  - **GitHub repository URLs, `git clone` instructions, and local
    clone-folder paths** (`docs/guides/*.md`, the site's Svelte/JS source,
    `src/crystalcore/README.md`, `AGENTS.md`'s title line, `mythos/README.md`'s
    and the two crystal-* `README.md`'s repo-slug labels) — these all mirror
    the actual GitHub repository name, `CrystalArchitect/TeraAustralis-Incognita`,
    which is unchanged and still double-a. Editing these to the corrected
    spelling now would make them wrong, not right — the URLs would 404 and
    the clone instructions would name a folder `git clone` doesn't create.
- **The GitHub repository name itself is out of scope for this ADR.**
  Renaming it is the maintainer's call, not this repository's: it breaks or
  redirects every external link, local clone, CI reference, and bookmark
  pointing at the current name, and (unlike a `git mv`) isn't reversible by
  a revert commit. If the maintainer wants to pursue it, GitHub redirects
  the old name for a period after a rename, but every doc listed above as
  "left alone" would need a follow-up pass once it happens. Recorded here
  as an open item, the same way [`ADR-0006`](ADR-0006.md#decision) recorded
  trademark registration as open rather than assuming it was done.

## Consequences

- The name printed everywhere a reader will actually see it — README,
  NOTICE, CONTRIBUTING, the Constitution, the vision docs, the mythos
  content itself — now matches the legal name it's supposed to represent,
  closing the exact gap [`ADR-0006`](ADR-0006.md) flagged as a precision
  risk for the trademark search that follows.
- `mythos/teraaustralis/` no longer exists as a path; anything outside this
  repo (bookmarks, external links into the old directory) breaks. Nothing
  inside the repo does — `git mv` preserved history and every internal
  reference was updated in the same change.
- The GitHub repo name (`TeraAustralis-Incognita`) and the project's actual
  name (`TerAustralis Incognita`) now visibly disagree by one letter. That
  mismatch was already latent — this ADR makes it legible instead of
  leaving it to be discovered mid-trademark-filing.
- `ADR-0001` and `ADR-0002` still say "TeraAustralis" in their historical
  text; a reader following those ADRs' own words to today's tree will find
  the directory renamed. Both now carry an implicit pointer forward via
  this ADR's presence in the index.
