# ADR-0006: Licensing strategy — keep the dual license, record the IP principles, flag trademark as unfinished

**Status:** Accepted
**Date:** 2026-07-23

## Context

The repository has carried a working dual license since before this ADR:
**Apache-2.0** for code (`src/`, `tests/`, `scripts/` — `LICENSE`) and
**CC BY-NC-ND 4.0** for content (`mythos/` — `LICENSE-CONTENT.md`),
documented in [`CONTRIBUTING.md`](../../CONTRIBUTING.md#licensing--read-before-reusing-content).
Neither file was chosen carelessly, but neither was chosen against a
written set of goals either — the split predates any explicit statement of
what the project wants licensing to *do*.

The maintainer surfaced six goals for CrystalCore OS and TeraAustralis
Incognita as a platform: the founder remains steward; contributors get
credit but no one may claim to be the original creator; the project names
and branding aren't misrepresented; outside contribution stays welcome
through the normal review path; a commercial future stays open rather than
being given away by accident; and major governance decisions stay outside
any single tool's hands. Per [`Decision-Records.md`](../governance/Decision-Records.md),
a decision this structural — and this hard to walk back once code has been
forked under a license — gets an ADR rather than a quiet file edit.

The [Incognita Rule](../governance/The-Incognita-Rule.md) applies here as
much as anywhere: this ADR is only allowed to assert what is actually true
today (a license file's terms, a governance doc that already exists). It
must not describe unregistered trademark protection, an unwritten CLA, or
any other legal step that hasn't happened as if it had.

## Decision

**1. Keep both license files as they are — no change to `LICENSE` or
`LICENSE-CONTENT.md`.** Checked against the six goals:

- *Commercial future*: CC BY-NC-ND 4.0 already blocks commercial use and
  derivatives of the mythos, art, and lore outright. Apache-2.0 on the code
  is a deliberate, different trade — it keeps the codebase a commons
  contributors can build on, in exchange for permitting commercial reuse of
  the *code* by anyone, including competitors. That is not a gap this ADR
  is closing; it is the standard shape of an open-core strategy (permissive
  code, restricted brand and content, commercial value sitting in the
  service/brand layer rather than in restricting the source). If that
  trade stops being acceptable, it is a future ADR that weighs
  Apache-2.0 against a source-available or copyleft alternative for
  `src/` specifically — not a default this one sets.
- *Attribution*: Apache-2.0 §4 and CC BY-NC-ND's attribution term already
  require it on redistribution.
- *Identity / no misrepresentation*: Apache-2.0 §6 already reserves all
  trademark, trade name, and service mark rights — a license grant is not
  a claim of authorship, and nothing in either file lets a redistributor
  call themselves the creator.
- *Collaboration*: unchanged — PR/review path in
  [`Review-Process.md`](../governance/Review-Process.md).
- *Stewardship* and *governance*: already binding, independent of any
  license file — the [Incognita Rule](../governance/The-Incognita-Rule.md)
  §3 ("no line mints its own authority... the human steward keeps the
  veto") and [`Repository-Principles.md`](../governance/Repository-Principles.md)
  already say this. A license controls redistribution rights; it was never
  the mechanism carrying stewardship, and this ADR doesn't make it one.

**2. Record the six goals as the project's licensing/IP principles**, so
future licensing questions get checked against a written standard instead
of instinct:

| Principle | Where it's actually enforced today |
|---|---|
| Stewardship | [Incognita Rule §3](../governance/The-Incognita-Rule.md#3-no-line-mints-its-own-authority); maintainer merge authority ([`Review-Process.md`](../governance/Review-Process.md)) |
| Attribution | Apache-2.0 §4; CC BY-NC-ND attribution term |
| Identity (naming/branding not misrepresented) | Apache-2.0 §6 (trademark reservation); [ADR-0004](ADR-0004.md) naming taxonomy |
| Collaboration | [`CONTRIBUTING.md`](../../CONTRIBUTING.md); [`Review-Process.md`](../governance/Review-Process.md) |
| Commercial future preserved | CC BY-NC-ND 4.0 NonCommercial term (content); open-core trade accepted for code (this ADR) |
| Governance stays outside any single tool | [`AI-Governance.md`](../governance/AI-Governance.md); Incognita Rule §3 |

**3. Trademark protection is not established and this ADR does not
establish it.** Apache-2.0 §6 reserves the *right* to control use of the
project's names and marks; it does not itself create trademark rights, and
no registration or formal trademark search has been done for "CrystalCore
OS" or "TeraAustralis Incognita." That is a legal step for the maintainer
to take outside this repository (use-in-commerce and, optionally,
registration with the relevant trademark office), not something a license
file or an ADR can grant. Recorded here as an open item, not a completed
one — consistent with the Incognita Rule's ban on dreamed lines dressed as
surveyed ones.

## Consequences

- Nothing in `LICENSE`, `LICENSE-CONTENT.md`, or existing contributor
  rights changes — no re-licensing, no new obligations on past
  contributions.
- The six principles now have one place they're checked against
  ([`CONTRIBUTING.md`](../../CONTRIBUTING.md#licensing--read-before-reusing-content)
  and [`docs/adr/README.md`](README.md) both point here), instead of living
  only in conversation.
- The commercial-code trade-off is now an explicit, recorded decision
  rather than an implicit side effect of having picked Apache-2.0 early —
  if it's ever revisited, the reasoning it's weighed against is written
  down.
- Trademark registration remains a real gap. It is the maintainer's to
  close (a legal/business step, not an engineering one) and does not block
  anything else in this ADR; it is flagged here so it isn't silently
  assumed to already be handled.
