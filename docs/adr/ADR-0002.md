# ADR-0002: Content areas — the mythos stays a top-level peer of docs and src

**Status:** Accepted
**Date:** 2026-07-23

## Context

The v1.0 blueprint names homes for code, docs, research, archive, assets,
and tests — but this repository also carries a large body of creative canon:
the Crystal universe (`mythos/`: Codex, Apocryphon, 88 art pieces, prompt
tools) and the outer-world lore (`TeraAustralis/`: mythic essence, publish
drafts). That material is none of the blueprint's categories: it is not
documentation of the software, not research toward it, and not archive. It
also sits under a different license (CC BY-NC-ND 4.0, `LICENSE-CONTENT.md`)
than the code (Apache-2.0), and the Built/Vision split that runs through
every honesty mechanism in the repo depends on the boundary staying sharp.

## Decision

- **`mythos/` remains a top-level area** — the one deliberate addition to
  the blueprint's root. Cramming the canon into `docs/` would misfile story
  as documentation and blur the license boundary; scattering it into
  `assets/` would treat canon as decoration.
- **`TeraAustralis/` folds into `mythos/teraaustralis/`** — one content
  root instead of two. Its strategy doc becomes
  `docs/vision/SouthernPillar.md` (that one *is* vision documentation, and
  the blueprint names it). The Constitution's reference to the publish
  folder is amended accordingly.
- **The art stays in `mythos/art/`**, not `assets/`. `assets/` is for
  project-level branding and diagrams; the art is licensed canon. The
  site's copies remain in `src/site/static/` because the build serves them.
- **Vision-layer code moves out of the content area**:
  `crystalcore_os.py` → `src/crystalcore-os/`. Content areas hold no
  executable code, in both directions of the code/docs rule.

## Consequences

- The license boundary is expressible in one sentence: everything under
  `mythos/` is content-licensed; everything under `src/` is code-licensed.
- `SystemMap.md` documents the deviation, so the blueprint and the tree
  don't silently disagree.
- The mythos terminal now lives apart from the canon it renders — its doc
  (`mythos/CRYSTALCORE-OS.md`) points across, which is the correct
  direction: story pointing at code.
