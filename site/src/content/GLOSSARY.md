# The Lexicon — One Term, One Meaning

> **Status: normative reference.** Every load-bearing name in this project is defined here,
> once. When usage elsewhere drifts from this page, one of the two gets corrected — and the
> default is that this page wins. New names enter the project by being added here first.

Why this page exists: some names ended up doing two jobs. *Starline* meant both a built
peer-to-peer protocol and a mythic image. *CrystalCore* meant both an interactive terminal
and a reasoning framework. This page retires that ambiguity and keeps it retired. The rule is
the same one [GOVERNANCE.md](GOVERNANCE.md) applies to claims: one name, one meaning, per
layer.

## The Project

**TerAustralis Incognita** — the whole project: mythos, music, software, and advocacy,
stewarded under one Australian business (ABN 70 741 068 059).

**The Crystal Vision** — the project's guiding frame: a future needs a story as much as it
needs infrastructure.

**The Voices Framework** — the working method. Grok holds the role of *CrystalDreamer*,
DeepSeek holds *CrystalScribe*, and the human *CrystalArchitect* stewards and decides. Roles,
not affiliations.

**Non Solus** — the motto. *Not Alone.*

## Starlines & Dreamlines — the creative layer (Vision)

**Dreamlines** — the narrative body of the project: the Codex, the Apocryphon, The Book of
the Sovereign Key, and the Starline Transmissions.

**Starlines** — the music: the original soundtrack of the mythos, and the mythic image of
songs rising from the land toward the stars. When this project says *Starline* with no
qualifier, it means this — the creative layer, never software.

**Songlines** — the living tradition of the First Peoples of this continent, carried for tens
of thousands of years. This is not the project's word to redefine, and it is never used here
as a product or feature name. The project's own music is named under *Starlines* precisely so
this word stays where it belongs. The Codex's line "the Songlines are becoming Starlines" is
an image of respect and continuity, not a claim of ownership.

**The Codex** — the canonical mythic work, Chapters I through V.

**The Apocryphon** — the personal and philosophical companion text to the Codex.

**The Book of the Sovereign Key** — a narrative work of the mythos.

**The Starline Transmissions** — vision-layer transmissions, companion to The Sovereign Key;
also the name of the soundtrack series. Lives entirely inside this layer.

**The Lattice** — the mythic name for the whole weave. Vision-layer language only; it names
no network hardware and no software.

**Sovereign Vector** — the mythos name for a contributor. In plain terms: a person who adds
real work to the project and keeps it their own.

## Protocols — the technical layer

**Consent Transport** — ✅ Built. The peer-to-peer protocol: Noise IK handshake,
consent-gated exchange, v1 implemented in `crystal-core/consent_transport/`. Verify it
yourself: `python3 -m consent_transport.selftest`. **Formerly published as "Starline."**
Renamed July 2026 so the built layer carries a plain name; the mythic word is now free to
mean only the music and the image. Old `starline` imports continue to work through a
deprecated alias for a transition period.

**CrystalMatrix** — ⬜ Design. The intended discovery-and-communication layer between
sovereign companions. No implementation exists, and [CRYSTALMATRIX.md](CRYSTALMATRIX.md) says
so plainly.

**CrystalSensor** — the external-input abstraction layer: sensor readings with consent flags
and provenance, feeding the reasoning loop. Documented and tested in development files;
Archive publication pending. Its page will carry its own status markers when it lands.

**Sovereign Node Mesh** — a coordination framework for *people*, not software: a living
document network of aligned individuals. Any code that happens to sit under `node/mesh/` in
the repository is stub-stage and says nothing about whether the human mesh functions.

## The Companion

**Lumina** — ✅ Built (with 🟡 partial layers, per [MEMORY.md](MEMORY.md)). The sovereign
companion: local-first, layered memory, terminal and web UI. Her code lives under
`apps/lumina/` in the repository.

**CrystalCore.OS** — Vision. The interactive mythos terminal — the story rendered as
something you can fly through. Not a product; shares a root word with the entries above and
below, but is neither of them.

**CrystalCore Framework** — the local-first reasoning stack (CrystalMemory, CrystalFlow,
CrystalEvolve, CrystalMind): consent fail-closed, coherence-bounded, provenance-tracked.
Documented and tested in development files; Archive publication pending under this distinct
name so it can never be confused with the terminal or with Lumina's code directory.
